# v139.2 hotfix — раскладка окна динамики метрик

Исправлена регрессия v139.1: блок «Пунктирный тренд / корреляция» мог вставляться перед всей боковой панелью управления, а не внутрь неё. Из-за этого CSS Grid воспринимал его как отдельную колонку, настоящие фильтры уходили вправо, а график и таблица оказывались внизу слева и выглядели очень мелкими.

Что изменено:

- `APP_VERSION = 139.2`;
- `index.html` подключает `app.js?v=139.2` и `style.css?v=139.2`;
- `manifest.json` синхронизирован на `v139.2`;
- вставка блока `trend-fit-control-v120` теперь происходит строго внутри `.topology-trend-controls-v106/.v91`;
- добавлен CSS-guard для окна мультивременной аналитики: левая колонка — управление, правая — график, пояснение и таблица;
- график получил больше полезного места и центрирование внутри правой панели.

Проверки:

- `node --check app.js`;
- синхронизация версий;
- отсутствие раннего `init().then(...)`;
- отсутствие известных регрессионных строк;
- проверка безопасной вставки `trend-fit-control-v120` внутри sidebar.
