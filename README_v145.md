# v145 — исправление графиков динамики метрик

Что изменено относительно v144:

1. В `data/topology/multiyear_metrics_by_year.json` пересчитан показатель `ate_total_count`.
   Теперь график «число АТЕ всего» считает только объекты административных слоёв с `area_km2 >= 200`.
   Малые городские / горсоветские полигоны и технические мелкие объекты больше не раздувают этот график.

2. В том же файле пересчитаны демографические поля динамики:
   - `total_population`,
   - `avg_population`,
   - `population_density`,
   - `urban_population`,
   - `rural_population`,
   - `urban_share`.

   Население теперь суммируется напрямую из актуальных `data/admin/admin_YEAR.geojson`, то есть соответствует тому, что видно в таблице атрибутов. Малые городские/горсоветские полигоны включаются в население, но не учитываются в счётчике «АТЕ всего» при площади <200 км².

3. Обновлены подписи в окне динамики метрик и таблицах:
   - «число АТЕ ≥ 200 км²»;
   - «АТЕ всего (≥200 км²)».

4. Диагностика пересчёта лежит в:
   - `docs/v145_multiyear_metrics_corrections.json`;
   - `docs/v145_multiyear_metrics_corrections.csv`.

Код проекта обновлён до `APP_VERSION = 145`, `index.html` подключает `app.js?v=145`.
