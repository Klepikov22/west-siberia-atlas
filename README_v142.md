# v142 — производные данные как пересобираемый кэш

## Главный принцип

Базовыми источниками считаются только исходные пространственные слои:

- `data/admin/admin_YYYY.geojson` — реконструкции АТЕ по датам;
- `data/railways/*` — железные дороги;
- `data/hydro/*` — гидрография;
- `data/natural_boundaries/reference/*` — природные reference-слои;
- `data/connectivity/reference/*` — reference-слои стоимостного графа связности.

Производные слои, метрики и графы рассматриваются как **rebuildable cache**. Их не следует править вручную: после исправления базового слоя они должны пересобираться скриптом.

## Что реализовано в v142

1. Добавлен скрипт:

```bash
python tools/rebuild_derivatives.py --boundary-levels
```

2. Пересобраны `data/admin_boundary_levels/admin_boundary_levels_YYYY.geojson`.

3. Границы уровней АТД теперь строятся из **фактических общих границ полигонов АТЕ**, а не из прямых графовых рёбер между центрами.

4. Береговая линия океана не попадает в эти слои, потому что используются только общие границы между двумя административными полигонами.

5. Добавлен `data/derived_manifest.json` с перечнем базовых источников, производных файлов и хэшей исходных admin-слоёв.

## Что остаётся следующим этапом

Та же архитектура должна быть постепенно распространена на:

- природно-морфологические сегменты границ;
- метрики природной приуроченности;
- топологический граф;
- стоимостной граф потенциальной связности;
- ЖД-метрики и плотности;
- многолетние таблицы и сводки.

В браузере желательно оставлять только лёгкую агрегацию по текущей выборке и фильтрам, а тяжёлые геометрические операции выполнять build-скриптом.
